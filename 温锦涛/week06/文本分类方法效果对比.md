## 三、消融实验

### 3.1 单变量消融矩阵

| 实验 | 变量变化 | Accuracy 变化 | 说明 |
|------|---------|-------------|------|
| CLS → Mean | 池化策略 | **-2.0%** | 均值池化丢失 [CLS] 位置的特殊语义 |
| CLS → Max | 池化策略 | **-4.0%** | 最大池化对噪声敏感，分类效果最差 |
| 不加权 → 加权 | Loss 加权 | **-0.5%** | 牺牲整体精度换取 Macro F1 +0.03 |
| 不加权 → 加权（证券 Recall）| Loss 加权 | **+0.19** | 证券 Recall 从 0.11 提升到 0.30 |
| Epoch 1 → 3 | 训练轮数 | **+5%** | 2-3 epoch 性价比最高 |
| Epoch 3 → 5 | 训练轮数 | **+0~1%** | 开始过拟合，val loss 上升 |
| 数据 1K → 5K | 数据量 | **+8%** | 数据量增加 5 倍，精度提升明显 |
| 数据 5K → 53K | 数据量 | **+8%** | 全量数据优势显著 |
| BERT 5K → SFT 5K | 方法 | **+4~6%** | 同等数据量下，SFT 利用预训练知识优势 |
| BERT 53K → SFT 5K | 方法 + 数据量 | **-1~3%** | SFT 用 9.4% 数据接近 BERT 全量效果 |
| r=4 → r=8 | LoRA Rank | **+2%** | r=8 性价比最优 |
| r=8 → r=16 | LoRA Rank | **+1%** | 边际收益递减 |

### 3.2 关键洞察

- **加权 Loss** 是少数类拯救者：Accuracy 仅降 0.5%，但证券类 Recall 翻 3 倍
- **数据量** 是 BERT 的命门：从 1K 到全量 53K，Accuracy 提升 16%
- **LoRA** 性价比极高：1.08M 参数（0.22%）即达接近全量微调效果

## 四、深入分析

### 4.1 数据效率：BERT vs SFT

```
BERT 53K:  ████████████████████████████████████████  59.5% Acc  (100% 数据)
SFT  5K:   ██████████████████████████████░░░░░░░░░░  58.0% Acc  (9.4% 数据)
           ↑ SFT 用 1/10 数据就接近 BERT 的效果
```

SFT 数据效率高的原因：

- **BERT** 分类头从零学习，53K 条数据同时承担"学习语义"和"学习分类边界"双重任务
- **Qwen2** 预训练阶段已有世界知识，5K 条仅需教会模型"输出格式"和"任务适配"

### 4.2 推理速度权衡

```
方法          单条推理 (GPU)   单条推理 (CPU)   可批量    部署复杂度
────────────────────────────────────────────────────────────
BERT          ~5 ms            ~50 ms           ✅       低
Zero-shot     ~60 ms           ~2000 ms         ✅       中
SFT (LoRA)    ~60 ms           ~500 ms          ✅       中
```

- BERT 推理速度是 SFT 的 10-12 倍，适合高吞吐生产场景
- SFT 虽慢，但在线场景可通过 batch 推理和量化（如 GPTQ/AWQ）显著提速
- Zero-shot 的 CPU 推理极慢，仅适合离线/pipeline 场景
