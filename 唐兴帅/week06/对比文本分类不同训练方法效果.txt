1. BERT 全量微调
适用条件：
数据充足：每个类别的标注数据大于等于 500 条。
性能敏感：对推理延迟要求极高，需要小于 100ms。
核心优势：
精度高且稳定：在有足够数据支撑的情况下，全量微调能最大程度挖掘 BERT 的特征提取能力，通常能达到该任务的“天花板”精度。
速度快：相比大语言模型（LLM），BERT 参数量小得多，推理速度极快，适合高并发实时系统。
劣势：
冷启动难：如果某个新类别只有几条数据，微调效果会很差，容易过拟合。
维护成本：每次新增类别都需要重新训练模型。
2. 大模型 Few-shot Prompt (少样本提示)
适用条件：
数据稀缺：每个类别的标注数据少于 100 条。
变动频繁：业务类别经常新增或调整。
核心优势：
泛化能力强：利用大模型（如 GPT-4, Qwen-Max 等）强大的语义理解能力，只需给几个例子（Prompt）就能学会新任务。
灵活性极高：不需要重新训练模型，修改 Prompt 中的示例即可适应新类别。
劣势：
成本高、速度慢：调用大模型 API 费用较高，且推理延迟通常在秒级，无法满足 <100ms 的实时性要求。
不稳定性：输出结果可能受 Prompt 写法影响较大，存在一定的随机性。
3. 轻量分类模型 (BERT-base / 蒸馏模型)
适用条件：
Agent 路由：用于智能体（Agent）的任务分发或意图识别。
高频调用：每秒查询率（QPS）大于 100。
核心优势：
性价比之王：在精度和速度之间取得了最佳平衡。蒸馏模型（如 DistilBERT, TinyBERT）保留了大部分 BERT 的能力，但体积更小、速度更快。
工程落地容易：非常适合部署在资源受限的边缘设备或需要处理海量请求的后端服务中。
定位：它是方案 1 的“轻量化”版本，专门针对工业界的高吞吐需求。
4. 大模型 Zero-shot (零样本) + 验证后收集标注
适用条件：
从 0 到 1：快速验证业务可行性（MVP 阶段）。
无标注数据：手头没有任何人工标注的数据集。
核心优势：
极速启动：完全不需要训练数据，直接利用大模型的常识知识进行分类。
数据飞轮起点：利用大模型的预测结果作为“伪标签”，经过人工校验后，转化为高质量的标注数据，为后续训练小模型（方案 1 或 3）积累资产。
劣势：
精度不可控：在没有领域特定知识的情况下，准确率通常低于微调模型。
仅适合过渡：随着业务发展，最终通常还是需要转向微调方案以降低成本。
