# RAG 基础

RAG 的全称是 Retrieval-Augmented Generation，中文通常称为检索增强生成。

RAG 的典型流程可以分为文档处理、检索和答案生成三个阶段。首先对原始文档进行解析与切分，然后为文本块建立检索索引。用户提出问题后，系统检索与问题最相关的内容，并将检索结果作为上下文用于回答。

文档切分会直接影响检索效果。文本块过大时容易包含无关信息，文本块过小时又可能破坏上下文完整性。实际系统通常需要根据文档类型设置合适的分块长度和重叠范围。

RAG 的优点是可以利用外部知识，并且知识库内容更新后不需要重新训练模型。对于企业内部制度、产品手册、课程资料等内容，RAG 是一种常见的问答系统实现方式。
