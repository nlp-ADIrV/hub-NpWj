# 大模型推理部署

大模型推理部署的主要目标是在有限计算资源下提高吞吐量、降低延迟并控制显存占用。

vLLM 通过 PagedAttention 等机制改进 KV Cache 的内存管理，并支持连续批处理，因此可以提升大模型服务的吞吐量。实际部署时还需要结合模型规模、GPU 显存、并发请求数量和上下文长度进行参数设置。

量化可以降低模型权重所需的显存。常见方法包括 INT8 和 INT4 量化，但量化程度越高，越需要关注模型精度变化。

在线服务通常需要关注首 token 延迟、单 token 延迟、吞吐量和并发能力。对于长文本请求，还需要重点观察 KV Cache 占用和最大上下文长度。
