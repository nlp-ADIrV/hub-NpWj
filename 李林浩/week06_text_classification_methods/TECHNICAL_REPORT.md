# 第六周作业技术报告：文本分类不同训练方法效果对比

## 一、实验背景

文本分类是自然语言处理中的基础任务之一，常见应用包括情感分析、新闻分类、垃圾邮件识别、舆情识别、意图识别等。虽然文本分类任务本身并不复杂，但模型最终效果不仅取决于模型结构，也与训练方法密切相关。

本次实验围绕“对比文本分类不同训练方法效果”这一主题展开。实验没有追求复杂的大模型或大规模数据集，而是选择构造一个规则清晰、边界明确的中文短文本二分类任务，在固定模型结构的前提下，对比不同优化器、损失函数、正则化策略和学习率调度方法对模型训练效果的影响。

这样的实验设计有一个好处：可以尽量减少模型结构和数据难度带来的干扰，更直接地观察训练方法本身的作用。

---

## 二、实验任务说明

### 2.1 任务定义

本实验设计了一个中文短文本二分类任务。模型输入为一句中文短文本，输出为二分类标签：

| 标签 | 含义 | 判断规则 |
|---|---|---|
| 1 | 正样本 | 文本中包含“好、棒、赞、喜欢、满意”等积极关键词 |
| 0 | 负样本 | 文本中不包含上述积极关键词 |

例如：

| 文本 | 标签 |
|---|---|
| 这家餐厅真的很好，下次还来 | 1 |
| 这款产品设计让我满意 | 1 |
| 下午开了三个小时的会议 | 0 |
| 公交车又晚点了十分钟 | 0 |

该任务属于典型的中文短文本分类任务。虽然它比真实业务中的文本分类任务更简单，但它适合作为训练方法对比实验，因为数据规则清楚、实验结果容易观察，能够较好地体现不同训练策略对模型收敛和泛化表现的影响。

---

## 三、数据集构造

### 3.1 数据来源

本实验没有使用外部公开数据集，而是通过代码自动生成训练数据。数据生成方式包括：

1. 正样本通过模板句构造，并随机插入积极关键词；
2. 负样本从中性或日常描述性语句中随机抽取；
3. 部分样本进行长度扩展或额外关键词插入，以增加数据多样性。

### 3.2 数据规模

| 项目 | 设置 |
|---|---|
| 样本总数 | 4000 条 |
| 训练集比例 | 80% |
| 验证集比例 | 20% |
| 训练集数量 | 约 3200 条 |
| 验证集数量 | 约 800 条 |
| 最大文本长度 | 32 |
| 编码方式 | 字符级编码 |
| 任务类型 | 二分类 |

本实验采用正负样本基本平衡的数据设置，因此 Accuracy 可以作为一个直观指标。同时，为了更全面地观察模型效果，实验也统计了 Precision、Recall、F1、FP、FN 等指标。

---

## 四、模型设计

### 4.1 模型选择

本实验固定使用一个轻量级 RNN 文本分类模型 `KeywordRNN`。之所以固定模型结构，是为了保证不同实验之间只有训练方法发生变化，从而使对比更加公平。

模型结构如下：

```text
输入文本
  ↓
字符级编码
  ↓
Embedding 层
  ↓
RNN 层
  ↓
Max Pooling
  ↓
BatchNorm
  ↓
Dropout
  ↓
Linear 分类层
  ↓
二分类输出
```

### 4.2 模型结构说明

**Embedding 层**  
将每个中文字符映射为连续向量表示，使模型能够处理离散文本输入。

**RNN 层**  
对字符序列进行建模，提取文本中的顺序信息和上下文特征。

**Max Pooling 层**  
从所有时间步的隐藏状态中提取最显著的特征。对于关键词触发型任务，最大池化能够较好地捕捉关键字带来的强信号。

**BatchNorm 层**  
对中间特征进行归一化，有助于提升训练稳定性。

**Dropout 层**  
随机丢弃部分神经元，降低模型对训练集的过度拟合风险。

**Linear 层**  
将文本特征映射为一个二分类 logit，用于判断文本是否属于正样本。

## 五、对比训练方法

本实验共对比 6 种训练方法。

| 编号 | 方法 | 优化器 | 损失函数 | 额外策略 |
|---|---|---|---|---|
| 1 | Adam + BCE | Adam | BCEWithLogitsLoss | 无 |
| 2 | SGD + BCE | SGD | BCEWithLogitsLoss | Momentum |
| 3 | Adam + MSE | Adam | MSELoss | 无 |
| 4 | AdamW + WeightDecay | AdamW | BCEWithLogitsLoss | 权重衰减 |
| 5 | Adam + LabelSmooth | Adam | Label Smoothing BCE | 标签平滑 |
| 6 | Adam + CosineLR | Adam | BCEWithLogitsLoss | 余弦学习率调度 |

## 六、评价指标

本实验使用以下指标评价模型效果：

| 指标 | 含义 |
|---|---|
| Accuracy | 所有样本中预测正确的比例 |
| Precision | 被预测为正样本的文本中，真正为正样本的比例 |
| Recall | 所有真实正样本中，被模型成功识别出来的比例 |
| F1 | Precision 和 Recall 的调和平均 |
| FP | 负样本被误判为正样本的数量 |
| FN | 正样本被误判为负样本的数量 |

仅看 Accuracy 有时会掩盖模型在正负样本上的不同表现。因此，本实验同时关注 Precision、Recall 和 F1。尤其在文本分类任务中，如果某一类更重要，就不能只看整体准确率，而要进一步分析错判类型。

## 七、实验过程

实验流程如下：

```text
1. 自动生成中文短文本数据
2. 构建字符级词表
3. 将文本编码为固定长度序列
4. 划分训练集和验证集
5. 固定 KeywordRNN 模型结构
6. 依次训练 6 种训练方法
7. 每个 epoch 输出 loss、accuracy、F1 和学习率
8. 保存每种方法的训练过程与最终结果
9. 对比不同训练方法的效果
```

程序运行后会生成 `results/summary_results.csv`，其中保存不同训练方法的最终指标；同时也会生成每种方法对应的 `history_*.csv`，用于观察训练过程。

## 八、实验结果分析

由于该任务的数据规则较为明确，模型通常能够较快学习到分类规律。不同训练方法之间的差异主要体现在以下几个方面：

1. 收敛速度是否足够快；
2. 验证集 Accuracy 和 F1 是否稳定；
3. 是否容易出现过拟合；
4. 错误类型主要集中在 FP 还是 FN；
5. 学习率变化是否改善训练后期表现。

### 8.1 Adam + BCE

Adam + BCEWithLogitsLoss 是本实验中最适合作为 baseline 的方法。

Adam 属于自适应学习率优化器，在训练神经网络时通常具有较好的稳定性。BCEWithLogitsLoss 是二分类任务中非常常用的损失函数，它将 sigmoid 和 binary cross entropy 合并计算，数值上更加稳定。

在本实验中，Adam + BCE 的优点主要是：

1. 收敛速度快；
2. 训练过程稳定；
3. 与二分类任务匹配度高；
4. 实现简单，适合作为默认方案。

因此，如果只选择一种基础训练方法，Adam + BCE 是最稳妥的选择。

### 8.2 SGD + BCE

SGD 是经典优化器，但它对学习率设置更加敏感。即使加入 Momentum，SGD 在相同训练轮数下也可能比 Adam 收敛更慢。

本实验中设置 SGD + BCE 的目的，是作为 Adam 的对照组。通过对比可以看到，自适应优化器在小型文本分类任务中通常更容易获得稳定结果，而 SGD 往往需要更多调参经验。

SGD 的价值不在于它一定效果最好，而在于它能帮助理解优化器选择对训练过程的影响。

### 8.3 Adam + MSE

MSELoss 原本更适合回归任务，而不是分类任务。对于二分类问题，模型本质上是在学习类别概率或类别边界，因此 BCEWithLogitsLoss 更符合任务目标。

在代码中，为了让 MSE 能够用于二分类，先将 logits 经过 sigmoid 转换为概率，再与标签计算均方误差。虽然这样模型也可以训练，但从理论匹配性上看，它不如 BCE 合理。

这个对照实验说明：损失函数不能随便选择。即使模型结构相同，损失函数与任务目标不匹配，也可能影响模型收敛和最终效果。

### 8.4 AdamW + Weight Decay

AdamW 可以看作 Adam 的改进版本，它将权重衰减与梯度更新更合理地解耦。Weight Decay 的作用是限制模型参数过度增大，从而降低过拟合风险。

在简单任务中，AdamW 不一定显著提升最高 Accuracy，但它在泛化稳定性上更有意义。对于更复杂的数据、更大的模型或更长的训练过程，AdamW 往往比普通 Adam 更值得优先考虑。

### 8.5 Adam + Label Smoothing

Label Smoothing 的核心思想是降低模型对标签的过度自信。普通二分类标签是 0 或 1，而标签平滑会将标签变得不那么绝对。

例如：

```text
原始正样本标签：1
平滑后标签：0.9

原始负样本标签：0
平滑后标签：0.1
```

这样可以让模型不要过度相信训练数据中的标签，尤其适合标签存在噪声的任务。

不过，本实验中的标签规则比较明确，噪声较少，因此 Label Smoothing 的提升可能不明显。它更适合真实场景中标注质量不稳定的数据。

### 8.6 Adam + CosineLR

CosineLR 使用余弦退火策略调整学习率。训练初期学习率较高，有利于快速下降；训练后期学习率逐渐降低，有利于模型更平稳地收敛。

在本实验中，CosineLR 的意义主要体现在训练后期。如果训练轮数较多，学习率调度往往比固定学习率更稳。它未必每次都能显著提升最高分，但能够改善训练过程的平滑性。

## 九、综合结论

通过本次实验，可以得到以下结论：

### 9.1 训练方法会显著影响模型效果

即使模型结构完全相同，只改变优化器、损失函数或学习率策略，也会影响模型的收敛速度、稳定性和最终指标。

这说明模型训练并不是简单地“把模型跑起来”，而是需要理解任务目标、损失函数、优化器和正则化方法之间的关系。

### 9.2 二分类任务应优先使用 BCEWithLogitsLoss

对于二分类任务，BCEWithLogitsLoss 比 MSELoss 更合理。  
MSE 虽然也能训练，但它与分类任务目标并不完全匹配。

因此，在实际项目中，损失函数应根据任务类型选择，而不是随意替换。

### 9.3 Adam 是较稳定的基础优化器

Adam 的收敛速度和稳定性通常优于普通 SGD，尤其适合快速建立 baseline。  
对于初学实验或小型文本分类任务，Adam 是一个很实用的默认选择。

### 9.4 正则化方法更关注泛化能力

AdamW、Weight Decay、Dropout、Label Smoothing 等方法的价值，不一定体现在训练集准确率上，而更多体现在泛化能力和训练稳定性上。

在简单数据集上，它们可能没有特别明显的提升；但在真实任务中，这些方法往往更加重要。

### 9.5 学习率调度可以改善训练后期表现

固定学习率不一定适合所有训练阶段。  
CosineLR 通过动态调整学习率，可以让模型在训练后期更加稳定地收敛。

## 十、方法选型建议

| 使用场景 | 推荐方法 | 理由 |
|---|---|---|
| 快速建立 baseline | Adam + BCE | 收敛快、实现简单、效果稳定 |
| 标准二分类任务 | Adam + BCE | 损失函数与任务高度匹配 |
| 防止过拟合 | AdamW + WeightDecay | 有助于提升泛化能力 |
| 标签存在噪声 | Adam + LabelSmooth | 降低模型过度自信 |
| 多轮训练 | Adam + CosineLR | 后期收敛更加平稳 |
| 教学对照实验 | SGD + BCE / Adam + MSE | 便于观察训练方法差异 |

## 十一、实验心得体会

通过这次作业，我对文本分类模型训练有了更具体的理解。

一开始容易认为，文本分类的关键主要在于模型结构，比如使用 RNN、LSTM、CNN、BERT 或更大的语言模型。但通过本次实验可以发现，在模型结构固定的情况下，训练方法本身也会对结果产生明显影响。优化器、损失函数、正则化和学习率策略并不是代码里的附属配置，而是模型训练过程中的核心组成部分。

本次实验中，我印象最深的是损失函数和任务目标之间的关系。二分类任务使用 BCEWithLogitsLoss 是比较自然的选择，而 MSELoss 虽然也能运行，但它原本是为回归任务设计的。这个对比让我意识到，模型训练不是只要 loss 能下降就可以，还要看这个 loss 是否真正符合任务目标。

另外，Adam 和 SGD 的对比也很直观。SGD 是经典方法，但它对学习率更加敏感；Adam 则更加稳定，适合快速得到一个可用结果。这让我理解了为什么实际项目中经常先使用 Adam 建立 baseline，再考虑其他更复杂的优化策略。

我也进一步理解了正则化方法的意义。像 AdamW、Weight Decay、Dropout、Label Smoothing 这些方法，不一定每次都能让 Accuracy 立刻变高，但它们的重点是让模型不要过度依赖训练数据，从而提升泛化能力。在真实任务中，数据通常不会像实验数据这样干净，因此正则化和鲁棒性会更加重要。

从工程角度看，这次作业也让我体会到实验记录的重要性。只输出一个最终准确率是不够的，更好的方式是记录每个 epoch 的 loss、accuracy、F1 和学习率，并把最终结果保存为 CSV 文件。这样后续才能复盘不同方法的训练过程，而不是只凭一次输出结果下结论。

总体来说，这次实验让我认识到，文本分类并不是简单地调用一个模型，而是一个完整的训练系统。数据构造、模型结构、损失函数、优化器、学习率策略、评价指标和结果分析都需要配合起来，才能形成一个完整的实验。相比只看模型最终分数，我现在更能理解每个训练配置背后的作用，也更清楚在不同任务场景下应该如何选择训练方法。

## 十二、后续改进方向

本实验仍然是一个相对基础的教学实验，后续可以从以下几个方向继续改进：

1. 将 RNN 替换为 LSTM、GRU、TextCNN，对比不同模型结构；
2. 引入真实数据集，例如新闻分类、情感分析或意图识别数据；
3. 增加类别不均衡实验，观察 Precision、Recall 和 F1 的变化；
4. 加入 Early Stopping，防止训练后期过拟合；
5. 增加训练曲线可视化，更直观地观察 loss 和指标变化；
6. 使用 BERT 等预训练模型作为更强 baseline；
7. 在小样本场景下对比传统模型、深度学习模型和大语言模型 Prompt 方法；
8. 加入混淆矩阵，进一步分析模型错分情况。

这些改进可以让实验更接近真实文本分类任务，也能更全面地理解不同方法在不同场景下的适用性。

## 十三、最终总结

本次作业完成了一个完整的中文短文本分类训练方法对比实验。实验固定使用 KeywordRNN 模型，通过对比 Adam、SGD、AdamW、MSE、Label Smoothing 和 CosineLR 等方法，分析了不同训练策略对模型效果的影响。

最终可以得到一个较清晰的结论：对于标准二分类任务，Adam + BCEWithLogitsLoss 是最稳妥的基础方案；AdamW 和 CosineLR 可以作为进一步优化训练稳定性和泛化能力的方案；MSELoss 更适合作为对照实验，而不适合作为二分类首选损失函数。

这次实验不仅完成了文本分类模型训练，也帮助我更系统地理解了“训练方法”在机器学习实验中的作用。一个好的文本分类实验，不只是模型能跑通，更重要的是能够解释为什么这样训练、不同方法有什么差异，以及在实际场景中应该如何选择。
